CGI,FastCGI,php-fpm

发表于 2017-09-19

首先了解相关概念

CGI

CGI是干嘛使得的？
场景：如果一个http请求发送给nginx服务器，nginx服务器在接受到这个请求之后，知道这个请求是一个动态的请求，那么这个请求就需要php来解析，动态的请求需要传入一些信息进来，这些信息是有对应的格式的，而CGI就是来规范这些格式的协议。
所以CGI就是一个协议，和进程没有关系！！！
CGI是为了保证web server传递过来的数据是标准格式，它是一个协议。

FastCGI

那么FastCGI又是什么东东呢？
FastCGI是CGI的升级版，也是一种协议。
FastCGI的特点是会在一个进程中依次完成多个请求，以达到高效的目的，大多数的FastCGI实现都会维护一个进程池。

php-fpm

php-fpm管理的是php_cgi，而不是管理的FastCGI协议，php-fpm实现了FastCGI协议的程序，用来管理FastCGI调起的进程。

php_CGI

只是解析php脚本的程序，就是PHP的解析器。

文件文件名的搜索

通常find不常用，因为速度慢，很操硬盘。一般我们都是先使用whereis或者locate来搜索，如果真的找不到才会用find来查找。
为什么whereis要比find速度快呢？
其实是whereis只是找几个特定的目录而已，主要是对/bin/sbin下面的可执行文件，以及/usr/share/man下面的文件，所以速度比较快。

locate命令：

常用参数说明：
-i：忽略大小写差异；
-l：仅输出几行的意思，例如五行 -l 5
使用locate实例：

locate使用限制？
因为locate是经由数据库来搜寻的，而数据库的创建是默认每天执行一次的，所以新创建的文件，在数据库没有更新之前是查询不到的，那么如何更新数据库呢？
更新locate的数据库非常简单，直接updatedb就可以，可能需要等待几分钟。

updatedb： 根据 /etc/updatedb.conf 的设置去搜寻系统硬盘内的文件名， 并更新/var/lib/mlocate 内的数据库文件。
locate： 依据 /var/lib/mlocate 内的数据库记载， 找出使用者输入的关键字文件名。

find命令：

最常用的用法，以及选项参数：
find / -name nginx
选项与参数：
-name filename：搜索文件名为filename的文件
-size [+-size]：搜寻比size还要大（+）或者小（-）的文件
-user name：按照用户账号名称查询
-group name：按照群组名查询
-mtime [+-n]：列出n天之前（+）之后（-）被改动多的文件名

find的特殊功能就是能进行额外的操作：
例如：find / -name php.ini -exec ls -l {} \;

{} 代表的是“由 find 找到的内容”，find 的结果会被放置到 {} 位置中；
-exec 一直到 \; 是关键字， 代表 find 额外动作的开始 （-exec） 到结束 （\;） ， 在这中间的就是 find 指令内的额外动作。
在本例中就是“ ls -l {} ”啰！因为“ ; ”在 bash 环境下是有特殊意义的， 因此利用反斜线来跳脱。

MaHua是什么?

一个在线编辑markdown文档的编辑器

测试图片显示

向Mac下优秀的markdown编辑器mou致敬

MaHua有哪些功能？

  • 方便的导入导出功能
    • 直接把一个markdown的文本文件拖放到当前这个页面就可以了
    • 导出为一个html格式的文件，样式一点也不会丢失
