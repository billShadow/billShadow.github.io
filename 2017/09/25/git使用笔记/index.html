git使用笔记

发表于 2017-09-25 | 分类于 Git |

用户相关配置：

git config --list --global //查询用户信息
git config --global user.name xiaobin //设置用户名，就是提交code的时候显示是谁提交的
git config --global user.email bill_xiaobin@163.com // 设置邮箱
git config --global --add user.name bill // 添加用户

git取别名

git config --global alias.co checkout // 取别名co
git config --global alias.ci commit //取别名ci

基础操作

git clone username@host://github.XXX <克隆出来之后的文件名>
git add <filename>/* // 提交单个或者多个文件，*提交所有文件到暂存区
git commit -m "提交说明" // 提交修改的文件
git push origin master // 提交到远程master分支上面

创建分支

git branch test // 创建test分支，并切换过去
git checkout -b develop origin master // 以远程master为主线创建分支

git checkout develop // 切换分支
git branch -a // 显示所有分支

合并操作

git merge --no-ff develop // 切换到master分支之后先pull下，然后合并开发分支即可push到远程

回退版本

git reset --hard commit_Id // 回滚到指定的提交点

如何查看commit_id
git log // 查询提交记录

如果有冲突如何处理：

可以先贮藏，然后显示出来
git stash // 贮藏操作，将之前修改的文件隐藏起来
git stash apply // 拉取成功之后，将贮藏的又显示回来

如何丢弃本地修改：
git checkout filename // 丢弃本地的修改，在为commit和add的情况下

标签 tag

发布一个版本时，我们通常需要在版本库中打一个标签。这样更加更加利于回滚
gut tag // 查看所有标签
git tag <name> // 用于创建一个标签，默认是HEAD,也可以指定commit_id

指定特定的标签信息
git tag -a <tagname> -m "标签说明"

查看指定标签的详细信息
git show <tagname>

删除分支

删除本地分支
git branch -d bugs // 删除本地bugs分支

删除远程分支
git branch -rd origin/bugs // 删除远程bugs分支

忽略特殊文件

有时候我们必须把文件放到Git工作目录中，但是不能提交他们，比如配置文件等等。每次git status都会显示，这个时候Git为我们提供这个这么一个文件.gitignore文件，然后把要忽略的文件名填进去，Git就会自动忽略这些文件。

忽略的原则：

  • 忽略操作系统自动生成的文件，比如缩略图，比如ide自动生成的文件.idea等；
  • 忽略编译生成的中间文件、可执行文件等。
  • 忽略自己的带有敏感信息的配置文件，比如存放的配置文件等。
